Figma Protoype Link 1

Prototype covers scenario 1, 2 & 3

Keypath Scenario 1 – New User Registration
  • Download app from App Store
  • Click on New User button
  • Add email and password and click on register button
  • When Terms & Conditions comes up, click on link to read terms
  • New user must agree to Terms and Conditions to continue
  • User will need to give permission to access Apple Health data (heart rate, emergency contacts, etc.)
  • Calibration required for app to collect baseline for facial cues
  • User to click on Start button when they commence their journey

Keypath Scenario 2 – Current User
  • Login using current email and password
  • User to click on Start button when they commence their journey

Keypath Scenario 3 – Journey exceeds time limit alert
  • User is already logged in and journey has commenced
  • Driver has been driving too long
  • User alerted to deactivate alarm
  • On success, Driver Aware will say "Pull over"
  • Driver pulls over and clicks on Continue to start rest timer
  • On conclusion of 15 minutes, user is requested to test their reaction/response time with a game
  • If user finished correctly then show reaction time and history and continue journey
  • If user does not succeed, rest timer is reset to 15 minutes again


Figma Prototype Link 2

Prototype covers scenario 4 only

Keypath Scenario 4 – Potential collision alert
  • User is already logged in and journey has commenced
  • Alert triggered in the event of rapid deceleration
  • User alerted
  • User to deactivate alert with voice command
  • If alert is not deactivated, notifications sent to Emergency contacts


Figma Prototype Link 3

Prototype covers scenario 5 only

Keypath Scenario 5 – Fatigue Cues alert
  • User is already logged in and journey has commenced
  • Data collected and measured to test for signs of fatigue (heart rate, movement, facial cues, etc.)
  • User alerted
  • User to deactivate alert with voice command
  • If alert is not deactivated, second alert is sent
  • If second alert is not deactivated then notifications sent to Emergency contacts
